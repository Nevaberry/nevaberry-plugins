# Security and Crypto Changes — RHEL 10

## Crypto Policy Changes

### DEFAULT Policy
- **RSA key exchange rejected** in TLS — use ECDH-based ciphers
- To re-enable for legacy systems: switch to LEGACY policy or apply custom subpolicy

### LEGACY Policy
- **SHA-1 signatures disallowed in TLS** (libraries other than OpenSSL)
- OpenSSL still accepts SHA-1 outside of TLS in LEGACY mode

### Algorithm Removals
- **DSA** removed from NSS (use RSA or ECDSA)
- **SEED** removed from NSS
- **HeartBeat** extension removed from TLS
- **SRP** authentication removed from TLS (incompatible with TLS 1.3)

## OpenSSL Changes

- **ENGINE API removed** — migrate to providers
  - `openssl-pkcs11` package removed
  - Use `pkcs11-provider` instead (uses new provider API)
- SHA-1 not accepted at `SECLEVEL=2` in TLS
  - If TLS 1.0/1.1 needed: set `SECLEVEL=0` + LEGACY policy
- `stunnel` no longer supports ENGINE API — use `pkcs11-provider`

## GnuTLS 3.8.9

- Certificate compression in TLS supported (RFC 8879)
- RSA-OAEP supported (RFC 8017)
- **RSA PKCS#1 v1.5** encryption/decryption **deprecated and disallowed by default**
- FIPS mode: PKCS#12 exports use PBMAC1 (RFC 9579) by default
- OCSP: all records in response now checked (not just first)
- Minimum RSA key size in FIPS mode: 2048 bits

## CA Certificate Trust Bundle

The trust bundle location has changed:

```bash
# Old path (no longer works the same way)
/etc/pki/tls/certs/ca-bundle.crt

# New path
/etc/pki/ca-trust/extracted/pem/tls-ca-bundle.pem
```

Update any software that references the old path directly.

## FIPS Mode

- `fips-mode-setup` command **removed**
- `/etc/system-fips` file **removed**
- Enable FIPS at install time: add `fips=1` to kernel command line
- Check FIPS status: `cat /proc/sys/crypto/fips_enabled`

## OpenSSH 9.9

- **Ed25519 keys** generated by default (except FIPS mode → RSA)
- `ChannelTimeout` keyword in `sshd_config` for inactive channel closure
- `EnableEscapeCommandline` option in `ssh_config`
- Agent key restriction system for forwarding controls
- FIDO improvements: `verify-required` cert option, reduced PIN prompts

## Post-Quantum Cryptography (Technology Preview)

System-wide crypto policies, OpenSSL, GnuTLS, NSS, and OpenSSH support PQ algorithms as a Technology Preview.

## Sequoia PGP

New PGP tools complement GnuPG:
- `sq` — Sequoia PGP CLI for key management, encryption, signing
- `sqv` — Sequoia signature verifier

## Keylime Agent 0.2.5

- IDevID/IAK support for device identity
- TLS 1.3 enabled by default
- `keylime-policy` tool replaces individual management scripts
- HTTP revocation webhooks removed — HTTPS required
